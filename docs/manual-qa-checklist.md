# Manual QA Checklist

## 목적
- `anglangl`의 현재 핵심 흐름을 브라우저에서 일관되게 검증하기 위한 체크리스트다.
- 이번 체크리스트는 `study material` 자산 재사용 흐름과 `dramaNlearn` 비동기 상태 UX에 집중한다.

## 사전 조건
- Django 서버 실행
- 로그인 가능한 테스트 계정 2개 준비
- Celery worker 실행
- Redis 실행
- PostgreSQL 또는 SQLite 중 현재 검증 대상 환경 준비

## 1. Study Material 재사용 흐름

### 1.1 생성 진입
- `/player/` 또는 `dramaNlearn/player/<id>/`로 이동한다.
- `학습 자료 만들기` 버튼이 보인다.
- 버튼 클릭 시 `/study/create/`로 이동한다.
- 소스 유형과 제목이 초안에 반영된다.

### 1.2 저장
- 제목, 생성 내용, 메모를 입력한다.
- 저장 후 `/study/<id>/` 상세로 이동한다.
- 상세 화면에서 자료 유형, 학습 목적, 난이도, 공개 상태가 보인다.
- `편집`, `공개로 전환` 또는 `비공개로 전환` 버튼이 보인다.

### 1.3 라이브러리 확인
- `/study/`에서 방금 만든 자료가 보인다.
- `보관 유형` 필터에서 `내가 만든 자료`로 보인다.
- `상세 보기`, `편집` 버튼이 동작한다.

### 1.4 공개 전환
- 상세 화면에서 `공개로 전환`을 누른다.
- 상세 화면의 공개 상태가 `공개`로 바뀐다.
- 공개 상태 설명에 `탐색 화면에 노출됨`이 보인다.

### 1.5 다른 사용자 탐색
- 다른 계정으로 로그인한다.
- `/study/explore/`로 이동한다.
- 공개 전환한 자료가 목록에 보인다.
- 작성자 정보가 보인다.
- `상세 보기`, `복제하기` 또는 `내 라이브러리로 복제`가 보인다.

### 1.6 복제
- 공개 상세에서 `내 라이브러리로 복제`를 누른다.
- 복제 후 내 라이브러리 상세로 이동한다.
- 자료가 `가져온 자료`로 표시된다.
- `복제 원본` 정보가 보인다.
- 공개 상태는 `비공개`로 저장된다.

### 1.7 재편집
- 복제한 자료에서 `편집`을 누른다.
- 생성 내용과 메모를 수정한다.
- 저장 후 상세에서 수정 내용이 반영된다.
- 원본 자료는 바뀌지 않는다.

## 2. DramaNlearn 비동기 상태 UX

### 2.1 URL 추가
- `/dramaNlearn/urls/`로 이동한다.
- 제목과 URL을 입력하고 `URL 추출 및 저장`을 누른다.
- 새 행이 추가되고 상태가 `queued` 또는 `fetching`으로 보인다.

### 2.2 상태 폴링
- 페이지 새로고침 없이 상태가 갱신되는지 확인한다.
- 작업 메시지와 진행률 퍼센트가 보이는지 확인한다.
- 성공 시 `ready` 상태와 재생 진입 링크가 유지되는지 확인한다.

### 2.3 실패 처리
- 의도적으로 잘못된 URL 또는 실패 가능한 케이스를 넣는다.
- 상태가 `error`로 바뀌는지 확인한다.
- 실패 메시지가 사용자에게 읽을 수 있는 형태로 보이는지 확인한다.
- `재시도` 버튼이 보이고 동작하는지 확인한다.

### 2.4 취소 처리
- `queued` 상태 작업에서 `취소` 버튼을 누른다.
- 상태가 `canceled`로 바뀌는지 확인한다.
- 취소된 작업이 더 진행되지 않는지 확인한다.

### 2.5 작업 이력
- `이력` 버튼으로 작업 이력 화면에 진입한다.
- 각 시도별 상태, 메시지, 오류가 구분되어 보이는지 확인한다.
- 플레이어 상세에서도 `작업 이력` 링크가 보이는지 확인한다.

## 3. 인증/권한 확인
- 비로그인 상태에서 `/study/`, `/dashboard/`, `/dramaNlearn/urls/` 접근 시 로그인으로 이동하는지 확인한다.
- 내 자료 상세는 다른 사용자가 직접 열 수 없는지 확인한다.
- 비공개 자료는 다른 사용자가 복제할 수 없는지 확인한다.
- `작업 이력`은 owner에게만 보이는지 확인한다.

## 4. 회귀 확인 포인트
- 네비게이션에서 `Library`, `Explore` 진입이 유지되는지 확인한다.
- `학습 자료 만들기` 진입 버튼이 `/player/`와 `dramaNlearn/player/` 양쪽에 계속 보이는지 확인한다.
- 공개/비공개 전환 후 목록 노출이 즉시 기대대로 바뀌는지 확인한다.
- imported 필터가 복제 자료만 정확히 보여주는지 확인한다.
